## Summary

Implement full CRUD module management with 6-rule server-side 2D grid validation for the Staccato API. Endpoints: GET pages/{id}/modules, POST pages/{id}/modules, PUT modules/{id}, DELETE modules/{id}, PATCH modules/{id}/layout. The service layer enforces minimum size, page boundary, overlap, content type, breadcrumb-empty, and title-uniqueness rules, throwing typed BusinessExceptions mapped to specific HTTP status codes and error codes.

## Technical Context

**Language/Version**: C# 13 / .NET 10
**Primary Dependencies**: ASP.NET Core 10, Entity Framework Core 10, AutoMapper, FluentValidation, System.Text.Json
**Storage**: SQL Server via EF Core 10 (InMemory provider for tests)
**Testing**: xUnit + Moq + WebApplicationFactory + InMemory EF
**Target Platform**: Windows/Linux server (ASP.NET Core Kestrel)
**Project Type**: Web service (ASP.NET Core WebAPI)
**Performance Goals**: Module create/update < 1s; PATCH layout < 500ms
**Constraints**: All validation server-side; no new libraries; domain layer pure (no EF/HTTP types)
**Scale/Scope**: Single-user per-request; overlap check O(n) against page modules (n typically < 50)
