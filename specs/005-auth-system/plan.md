# Implementation Plan: Authentication System

**Branch**: `005-auth-system` | **Date**: 2026-03-08 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-auth-system/spec.md`

## Summary

Implement a complete authentication system for the Staccato API with five endpoints (`POST /auth/register`, `POST /auth/login`, `POST /auth/google`, `POST /auth/refresh`, `DELETE /auth/logout`). Uses the existing JWT Bearer + HttpOnly cookie refresh token architecture already scaffolded in `Application`. New work: `IAuthService` + `AuthService` (Domain), `IJwtService` + `JwtService` (Application), `IGoogleTokenValidator` + `GoogleTokenValidator` (Application), `IPasswordHasher` + `BcryptPasswordHasher` (Application), two new `BusinessException` subclasses, ApiModels DTOs + FluentValidation validators with localization, a localization-aware update to `BusinessExceptionMiddleware`, and a full unit + integration test suite.

> **Note on user hints**: The user suggested adding `Google.Apis.Auth` to Domain and `BCrypt.Net-Next` to Domain. Per constitution Principle I (Domain → DomainModels only as project reference) and the clean-architecture goal of keeping external service dependencies out of Domain, both NuGets are placed in **Application** behind `IPasswordHasher` and `IGoogleTokenValidator` interfaces defined in Domain. The `JwtService` implementation is similarly placed in Application. See `research.md §Decisions 1–3`.

## Technical Context

**Language/Version**: C# 13 / .NET 10
**Primary Dependencies**: ASP.NET Core 10, Entity Framework Core 10, AutoMapper, FluentValidation, `Microsoft.AspNetCore.Authentication.JwtBearer` (existing), `Google.Apis.Auth` (new → Application), `BCrypt.Net-Next` (new → Application), `Microsoft.Extensions.Localization.Abstractions` (new → ApiModels)
**Storage**: SQL Server via EF Core — existing `Users` and `RefreshTokens` tables, no schema changes
**Testing**: xUnit, Moq, `Microsoft.AspNetCore.Mvc.Testing` with InMemory EF
**Target Platform**: ASP.NET Core 10 Web API (Windows/Linux server)
**Project Type**: Web service — layered monolith (Service / Repository / UoW)
**Performance Goals**: Auth endpoints respond within 500 ms under normal single-user load (spec SC-003)
**Constraints**: 10 req/min/IP rate limit on `/auth/*` (already wired); refresh token rotation on every renewal; theft detection revokes all user tokens
**Scale/Scope**: Single-tenant API; unlimited concurrent sessions per user; tokens retained in DB (not purged on revoke)

## Exception Subclass Pattern

Both new exceptions follow the existing pattern from `BusinessException`:

```csharp
// UnauthorizedException — two overloads required:
// • Generic: for future use; code defaults to "UNAUTHORIZED"
// • Code-specific: used by AuthService for INVALID_CREDENTIALS, NO_PASSWORD_SET, etc.
public class UnauthorizedException : BusinessException
{
    public UnauthorizedException(string code, string message, object? details = null)
        : base(code, message, details) { StatusCode = 401; }
    public UnauthorizedException(string message, object? details = null)
        : base("UNAUTHORIZED", message, details) { StatusCode = 401; }
    public UnauthorizedException() : this("UNAUTHORIZED", "Authentication failed.") { }
}

// ServiceUnavailableException
public class ServiceUnavailableException : BusinessException
{
    public ServiceUnavailableException(string message, object? details = null)
        : base("SERVICE_UNAVAILABLE", message, details) { StatusCode = 503; }
    public ServiceUnavailableException()
        : this("An external service is temporarily unavailable. Please try again later.") { }
}
```

## BusinessExceptionMiddleware Localization

The existing middleware is modified to accept `IStringLocalizer<BusinessErrors>`:

```csharp
public sealed class BusinessExceptionMiddleware(RequestDelegate next, IStringLocalizer<BusinessErrors> localizer)
{
    public async Task InvokeAsync(HttpContext context)
    {
        try { await next(context); }
        catch (BusinessException ex)
        {
            context.Response.StatusCode = ex.StatusCode;
            context.Response.ContentType = "application/json";
            var localizedMessage = localizer[ex.Code].ResourceNotFound ? ex.Message : localizer[ex.Code].Value;
            var payload = new { code = ex.Code, message = localizedMessage, details = ex.Details };
            await context.Response.WriteAsync(JsonSerializer.Serialize(payload));
        }
    }
}
```

## Cookie Helper

A private helper method in `AuthController`:

```csharp
private void SetRefreshCookie(string token, DateTime expiry) =>
    Response.Cookies.Append("staccato_refresh", token, new CookieOptions
    {
        HttpOnly = true,
        SameSite = SameSiteMode.Strict,
        Secure = !HttpContext.RequestServices
            .GetRequiredService<IWebHostEnvironment>().IsDevelopment(),
        Expires = expiry
    });

private void ClearRefreshCookie() =>
    Response.Cookies.Delete("staccato_refresh");
```
