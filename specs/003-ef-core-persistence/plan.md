## Summary

Implement all 12 EF Core entity classes in the `EntityModels` project and the complete database configuration layer in the `Persistence` project. This establishes the data persistence foundation for the entire application: `AppDbContext` with 12 `DbSet<T>` registrations, one `IEntityTypeConfiguration<T>` per entity (table names, PKs, FKs, unique indexes, filtered indexes, JSON column types), a `DbInitializer` invoked synchronously at startup via `Program.cs`, and three seeders for instruments, guitar chords (loaded from `guitar_chords.json`), and system style presets. The `PdfExport` domain model is also extended with a `LessonIds` property to keep domain and entity in sync.

## Implementation Notes

### Entity Classes (EntityModels)

Each entity mirrors its corresponding domain model with identical scalar property names and types, plus EF navigation properties. Navigation collections use `ICollection<T>` (not `List<T>` or `IEnumerable<T>`). No data annotations on entity classes — all configuration is via Fluent API in the `Configurations/` folder.

Key patterns:
- All `string` properties must be non-nullable unless they can genuinely be null (e.g., `PasswordHash`, `GoogleId`, `AvatarUrl`, `BlobReference`, `LessonIdsJson`).
- Navigation properties are non-nullable when the FK is required (e.g., `UserEntity User` on `RefreshTokenEntity`). Use `= null!` initialization to satisfy nullable analysis without constructor params.

### AppDbContext

```csharp
public class AppDbContext(DbContextOptions<AppDbContext> options) : DbContext(options)
{
    public DbSet<UserEntity> Users => Set<UserEntity>();
    // ... 11 more DbSet properties

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.ApplyConfigurationsFromAssembly(typeof(AppDbContext).Assembly);
    }
}
```

Uses primary constructor syntax (C# 13). `DbSet<T>` properties use expression-body `=> Set<T>()` pattern.

### EF Core Configurations

Each configuration class implements `IEntityTypeConfiguration<TEntity>` and is discovered automatically by `ApplyConfigurationsFromAssembly`. Key configuration patterns:

**Filtered unique index (GoogleId)**:
```csharp
builder.HasIndex(u => u.GoogleId)
    .IsUnique()
    .HasFilter("[GoogleId] IS NOT NULL");
```

**Partial unique index (active PdfExports)**:
```csharp
builder.HasIndex(e => e.NotebookId)
    .IsUnique()
    .HasFilter("[Status] = 0 OR [Status] = 1");
```

**JSON column type**:
```csharp
builder.Property(e => e.ContentJson)
    .HasColumnType("nvarchar(max)")
    .IsRequired();
```

**Composite unique index**:
```csharp
builder.HasIndex(e => new { e.NotebookId, e.ModuleType }).IsUnique();
```

**Restrict delete (Chord → Instrument)**:
```csharp
builder.HasOne(c => c.Instrument)
    .WithMany(i => i.Chords)
    .HasForeignKey(c => c.InstrumentId)
    .OnDelete(DeleteBehavior.Restrict);
```

### guitar_chords.json

A flat JSON array. Each element maps to one `ChordEntity` row:
```json
[
  {
    "name": "A",
    "suffix": "major",
    "positions": [ { "label": "...", "baseFret": 1, "barre": null, "strings": [...] } ]
  }
]
```

File must be copied to output directory. `Persistence.csproj` needs:
```xml
<ItemGroup>
  <Content Include="Data\guitar_chords.json">
    <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
  </Content>
</ItemGroup>
```

### Seeders

All seeders receive `AppDbContext` via constructor injection. Each exposes a single `SeedAsync(CancellationToken ct = default)` method.

**InstrumentSeeder**: Check `if (!await context.Instruments.AnyAsync(ct))` then insert 7 rows.

**ChordSeeder**:
1. Resolve `guitar_chords.json` relative to `AppDomain.CurrentDomain.BaseDirectory`.
2. If file not found → `throw new InvalidOperationException($"Chord seed file not found: {path}")`.
3. `JsonSerializer.Deserialize<List<ChordSeedRecord>>` — if null/exception → throw with file path in message.
4. Check `if (!await context.Chords.AnyAsync(ct))` then insert all rows.

**SystemStylePresetSeeder**: Check `if (!await context.SystemStylePresets.AnyAsync(ct))` then insert 5 presets with full color data from research.md Decision 4.

### DbInitializer

```csharp
public class DbInitializer(
    AppDbContext context,
    InstrumentSeeder instrumentSeeder,
    ChordSeeder chordSeeder,
    SystemStylePresetSeeder presetSeeder)
{
    public async Task InitializeAsync(CancellationToken ct = default)
    {
        await context.Database.MigrateAsync(ct);
        await instrumentSeeder.SeedAsync(ct);
        await chordSeeder.SeedAsync(ct);
        await presetSeeder.SeedAsync(ct);
    }
}
```

### AddDatabase Registration and Program.cs

**ServiceCollectionExtensions.cs** — `AddDatabase` method:
```csharp
public static IServiceCollection AddDatabase(
    this IServiceCollection services,
    IConfiguration configuration)
{
    services.AddDbContext<AppDbContext>(options =>
        options.UseSqlServer(configuration.GetConnectionString("DefaultConnection")));

    services.AddScoped<InstrumentSeeder>();
    services.AddScoped<ChordSeeder>();
    services.AddScoped<SystemStylePresetSeeder>();
    services.AddScoped<DbInitializer>();

    return services;
}
```

**Program.cs** — after `var app = builder.Build()`, before `app.Run()`:
```csharp
using (var scope = app.Services.CreateScope())
{
    var initializer = scope.ServiceProvider.GetRequiredService<DbInitializer>();
    await initializer.InitializeAsync();
}
```
