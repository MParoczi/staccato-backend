# Implementation Plan: Lesson & Lesson Page Management

**Branch**: `009-lesson-page-management` | **Date**: 2026-03-29 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/009-lesson-page-management/spec.md`

## Summary

Implement CRUD endpoints for lessons and lesson pages, plus an auto-generated notebook index endpoint. Lessons are scoped to notebooks, pages are scoped to lessons, and ownership is enforced through notebook-level user verification. The data model (entities, domain models, EF configurations, repository interfaces, repository implementations, AutoMapper entity↔domain profiles, DI registrations) already exists. This feature adds the **service layer**, **API models**, **controller layer**, and **response mappings** on top of the existing foundation.

## Technical Context

**Language/Version**: C# 13 / .NET 10
**Primary Dependencies**: ASP.NET Core 10, Entity Framework Core 10, AutoMapper, FluentValidation, BCrypt.Net-Next
**Storage**: SQL Server via EF Core 10 (tables `Lessons`, `LessonPages` already defined)
**Testing**: xUnit + Moq + WebApplicationFactory + InMemory EF
**Target Platform**: Windows/Linux server
**Project Type**: Web service (ASP.NET Core WebAPI)
**Performance Goals**: Standard web-app expectations; no special latency targets
**Constraints**: All patterns must follow existing Service/Repository/UoW architecture
**Scale/Scope**: Single-user access per request; no concurrency conflicts expected for page creation
